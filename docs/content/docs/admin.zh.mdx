FoxBlaze 基于 NestJS 框架动态构建，旨在通过 PM2 在 Linux 服务器上高效运行。

## 系统要求

在部署机器人之前，请确保您的服务器安装了以下系统依赖项：
- **Node.js**（推荐 v20+）
- **PostgreSQL**（Prisma 钱包和状态持久化所需）
- **Redis**（BullMQ 异步任务执行所需）
- **PM2**（用于后台进程管理）

## 应用程序设置

1. **克隆存储库**并安装 Node 依赖项：
```bash
git clone https://github.com/noweieth/foxblaze-trade.git
cd foxblaze-trade
npm install
```

2. **数据库生成**：初始化 Prisma schema 并应用数据库迁移。
```bash
npx prisma generate
npx prisma db push
```

## 环境配置

在根目录中创建一个 `.env` 文件。您必须提供您的 Telegram Bot 令牌和用于派生所有用户代理钱包的主助记词。

```bash
TELEGRAM_BOT_TOKEN=your_bot_token_from_botfather

# PostgreSQL Connection
DATABASE_URL="postgresql://user:pass@localhost:5432/foxblaze?schema=public"

# Redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# Cryptography (BIP-39 Master Key)
MASTER_SEED_PHRASE="your twelve words secret phrase goes here"
AES_SECRET_KEY="32_byte_aes_encryption_key_here"

# Native EVM RPC
ARBITRUM_RPC_URL="https://arb1.arbitrum.io/rpc"
```

<Callout title="安全警告" type="error">
必须以极高的安全级别对待 `MASTER_SEED_PHRASE` 和 `AES_SECRET_KEY`。如果这些密钥泄露，整个非托管结构将受到破坏。
</Callout>

## 部署 (PM2)

配置完成后，使用 PM2 构建并启动服务器：

```bash
npm run build
pm2 start dist/main.js --name foxblaze
pm2 save
```

使用 `pm2 logs foxblaze` 监控实时执行日志、WebSocket 监听器稳定性和 BullMQ 工作进程状态。
