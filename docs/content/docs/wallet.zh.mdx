FoxBlaze 是一个完全建立在 Hyperliquid L1 应用链上的**非托管**交易引擎。为了在 Telegram 中提供无缝的“免 Gas”交易体验，它使用**代理密钥 (Agent Keys)** 运行。

## 您的资金如何保持安全

当您使用 `/start` 命令时，机器人会从我们服务器上的安全主助记词中派生出为您生成的标准 **HD 钱包 (BIP-39/BIP-44)**。

### 1. AES-256 加密
生成的私钥绝不会以明文形式存储。它在插入数据库之前会立即使用 AES-256 进行加密。

### 2. 代理密钥系统
FoxBlaze **不会**使用您主钱包的私钥签名进行交易。相反，它会生成一个专门针对机器人的二级**代理密钥**。
在初始设置期间，您的主钱包会签署一次性交易（需要在 Arbitrum 上存入至少 10 USDC），以授权这个代理密钥。

从那时起，机器人仅持有并使用**代理密钥**，该密钥拥有代表您执行交易的权限。
- 代理密钥**可以**开仓和平仓，以及下达条件订单 (TP/SL)。
- 代理密钥**不能**将您的资金提取回 L1 (Arbitrum) 或外部地址。

这确保了即使机器人的临时内存遭到破坏，您的资金也会永久锁定在实际的钱包地址中，只有您控制其恢复短语（如果导出，*功能即将推出*）。

## 高性能执行

通过利用 Hyperliquid API 的代理(Agent)流程，FoxBlaze 直接在 Hyperliquid 网络上执行所有交易：
- **零 Gas 费**：无需原生 EVM 代币（如 ETH）即可下达订单。
- **微秒级延迟**：订单以中心化交易所的速度执行，同时保留去中心化、非托管结算的约束。
