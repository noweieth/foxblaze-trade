FoxBlaze динамично построен на фреймворке NestJS и предназначен для эффективной работы на Linux-сервере с использованием PM2.

## Требования к системе

Перед развертыванием бота убедитесь, что на вашем сервере установлены следующие системные зависимости:
- **Node.js** (рекомендуется v20+)
- **PostgreSQL** (для кошелька Prisma и хранения состояний)
- **Redis** (для асинхронного выполнения задач BullMQ)
- **PM2** (Менеджер процессов)

## Установка приложения

1. **Клонируйте репозиторий** и установите зависимости:
```bash
git clone https://github.com/noweieth/foxblaze-trade.git
cd foxblaze-trade
npm install
```

2. **База данных**: Инициализируйте схему Prisma и примените миграции.
```bash
npx prisma generate
npx prisma db push
```

## Конфигурация среды

Создайте файл `.env` в корневом каталоге. Вы должны указать ваш токен Telegram бота и мастер-фразу (seed) для деривации кошельков.

```bash
TELEGRAM_BOT_TOKEN=your_bot_token_from_botfather

# PostgreSQL Connection
DATABASE_URL="postgresql://user:pass@localhost:5432/foxblaze?schema=public"

# Redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# Cryptography (BIP-39 Master Key)
MASTER_SEED_PHRASE="your twelve words secret phrase goes here"
AES_SECRET_KEY="32_byte_aes_encryption_key_here"

# Native EVM RPC
ARBITRUM_RPC_URL="https://arb1.arbitrum.io/rpc"
```

<Callout title="Предупреждение безопасности" type="error">
К `MASTER_SEED_PHRASE` и `AES_SECRET_KEY` следует относиться с высочайшим уровнем безопасности. В случае утечки этих ключей, вся некастодиальная структура будет скомпрометирована.
</Callout>

## Развертывание (PM2)

После настройки соберите и запустите сервер:

```bash
npm run build
pm2 start dist/main.js --name foxblaze
pm2 save
```

Используйте `pm2 logs foxblaze` для мониторинга логов в реальном времени.
